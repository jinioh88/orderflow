## 1. 앱 셸 (App Shell)

```
┌─────────┬──────────────────────────────────────┐
│         │ 헤더 h=56 — 페이지 타이틀 · (우측) 알림 · 계정 │
│ 사이드바 ├──────────────────────────────────────┤
│ w=240   │ 콘텐츠 — 페이지 배경 pageBg,             │
│         │ 좌우 패딩 24, 최대폭 제한 없음(그리드 우선)   │
└─────────┴──────────────────────────────────────┘
최소 뷰포트 1280px. 반응형 없음 — 1280 미만은 가로 스크롤 허용.
```

- **사이드바**: `surface` 배경 + 우측 `border`. 상단 로고 영역(h=56, 테넌트명 표시),
  메뉴 8개, 하단에 사용자 이름·역할 + 로그아웃.
- **메뉴 항목**: h=36, 아이콘 20 + 라벨(web.body-strong). 기본 `textBody`,
  호버 `neutral-100` 배경, **활성 = `primaryBg` 배경 + `primary` 텍스트/아이콘 + 좌측 3px `primary` 바**.
- **메뉴 구성** (화면 인벤토리 순, 도메인 아이콘 매핑 사용):

  | 메뉴 | 아이콘(Lucide) | 마일스톤 |
  |------|------|:---:|
  | 대시보드 | `bar-chart-3` | M2 |
  | 수주 관리 | `clipboard-list` | M3 |
  | 출하 지시 | `truck` | M2~3 |
  | 상품 | `tag` | M1 |
  | 클레임 | `file-warning` | M4 |
  | 정산·미수금 | `wallet` | M4 |
  | 가맹점 | `store` | M1 |
  | 계정 관리 | `users` | M1 |

  미구현 마일스톤의 메뉴는 숨긴다 (disabled로 두지 않는다 — 데모에서 미완성처럼 보임).
- **헤더**: `surface` 배경 + 하단 `border`. 좌측 페이지 타이틀(web.display).
  우측: 알림 벨(NTF-03, v1.1 — 그 전까지 미노출) · 사용자 메뉴.
- 콘텐츠 스크롤 시 헤더 고정(sticky).

## 2. 버튼

| 변형 | 배경/보더 | 텍스트 | 용도 |
|------|------|------|------|
| primary | `primary` → 호버 `primaryHover` | 흰색 | 화면당 1개 원칙 — 주 액션(저장, 승인) |
| secondary | `surface` + `borderStrong` 보더 | `textBody` | 보조 액션 (취소, 다운로드) |
| ghost | 투명 → 호버 `neutral-100` | `textBody` | 그리드 인라인 액션, 아이콘 버튼 |
| danger | `feedback.danger.solid` | 흰색 | 거절, 삭제 — 확인 다이얼로그와 세트 |
| danger-ghost | 투명 → 호버 `danger.bg` | `danger.text` | 그리드 인라인 거절 |

- 크기: **md h=32**(기본) / sm h=28(그리드 인라인) / lg h=36(빈 상태 CTA, 로그인).
  패딩 좌우 12(sm 8), radius-md(6), 라벨 web.body-strong(13/600).
- 아이콘+라벨 시 아이콘 16, 간격 6. 아이콘 단독 버튼은 32×32 (툴팁 필수).
- 로딩: D2 §2.2 — 버튼 내 스피너 16, 라벨 유지, 폭 고정.
- disabled: 배경 `neutral-100`, 텍스트 `textDisabled` (opacity 트릭 금지 — 대비 예측 불가).

## 3. 인풋 · 셀렉트 · 날짜 범위

- **공통**: h=32, radius-sm(4), `surface` 배경, `borderStrong` 보더, 텍스트 web.bodyMd(14),
  플레이스홀더 `textDisabled`.
  포커스: 보더 `primary` + 외곽 3px `rgba(37,99,235,0.15)` 링.
  에러: 보더 `danger.solid` + 포커스 링도 danger 15%. 라벨·검증 메시지 규칙은 D2 §3.
- **단위 서픽스** (수량·금액·한도): 인풋 내부 우측에 `textCaption` 색으로 고정 (`원`, `박스`).
  값은 우측 정렬 + tabular.
- **셀렉트**: 인풋과 동일 몸체 + 우측 `chevron-down` 16. 드롭다운은 shadow-2, 옵션 h=32,
  선택 옵션 `primaryBg` 배경 + 체크 아이콘.
- **검색 인풋**: 좌측 `search` 아이콘 16, 입력 시 우측 clear 버튼. 그리드 필터바 전용 폭 240.
- **날짜 범위**: 프리셋 칩(`오늘` `7일` `30일` `직접 지정`) + 직접 지정 시 캘린더 팝오버.
  선택 결과 표기 `2026-07-01 ~ 2026-07-19`. 미래 날짜는 조회 화면에서 비활성.

## 4. 데이터 그리드 (AG-Grid) — US-RCV-02의 중심 컴포넌트

D0 결정 #6: Quartz 테마 위에 토큰만 오버라이드. 커스텀 CSS는 아래 항목으로 한정한다.

### 4.1 테마 오버라이드 값

| AG-Grid 파라미터 | 값 |
|------|------|
| rowHeight / headerHeight | 36 / 32 |
| 폰트 | Pretendard, 셀 web.body(13) · 헤더 web.caption(12/500) `textCaption` |
| 헤더 배경 | `neutral-50`, 하단 보더 `borderStrong` |
| 행 보더 | 수평선만 `border` (수직선 없음 — 밀도 확보) |
| 짝수 행 | 줄무늬 없음. **호버 `neutral-50` / 선택 행 `gridRowSelected`** |
| 선택 체크박스 | 좌측 고정 컬럼 w=40, `primary` |
| 패딩 | 셀 좌우 12 |

### 4.2 컬럼 규칙

- 숫자(수량·금액)는 **우측 정렬 + `web.num`(tabular)**, 헤더도 우측 정렬. 포맷은 파운데이션 §5.
- 상태 컬럼: **badge-sm** (D2 §1). 필터는 상태 다중 선택.
- 날짜 컬럼: `YYYY-MM-DD HH:mm`, `textCaption` 색.
- 액션 컬럼(우측 고정): sm ghost 버튼 — 수주 목록은 `승인`(ghost) + `거절`(danger-ghost) 상시 노출
  (호버 노출 금지 — 키보드/스캔 효율). 낙관적 업데이트 동작은 D2 §4.2.
  **v1.1 전환 계획** (2026-07-19 외부 리뷰 반영): US-RCV-04 일괄 승인 도입 시
  상단 일괄 처리 액션 바를 주 동선으로 승격하고, 인라인 버튼은 호버 노출로 전환해
  시각 복잡도를 낮춘다. MVP에서는 일괄 처리가 없으므로 상시 노출을 유지한다.
- 긴 텍스트(사유·메모)는 1줄 말줄임 + 툴팁.

### 4.3 필터바 & 푸터

```
[프리셋: 승인 대기 | 여신 초과 | 한정 품목]  ← 필터 프리셋 칩 (자주 쓰는 조회 시나리오)
[검색 인풋 240] [상태 ▾] [가맹점 ▾] [날짜 범위]        (우측) 총 1,234건 · [↓ 엑셀 다운로드]
─────────────────────────────────────────────────────────────────
                         (그리드 — Virtual Scroll, 페이지네이션 없음)
```

- 필터바는 그리드 위 한 줄(h=48), 적용 즉시 반영(적용 버튼 없음). 활성 필터는 값이 라벨에 표시.
- **필터 프리셋 칩** (2026-07-19 외부 리뷰 반영): 김운영의 루틴 조회 시나리오를 원클릭으로.
  필터바 위(또는 좌측)에 칩 형태, 클릭 시 해당 필터 조합을 일괄 적용(기존 필터 대체).
  화면별 프리셋 구성은 각 화면 정의서에서 확정 — 수주 목록(M3)은
  `승인 대기` `여신 초과` `한정 품목 포함`, 미수금(M4)은 `60일 이상` `한도 초과` 를 기본안으로 한다.
  프리셋은 MVP에서는 고정 제공(사용자 정의 저장은 범위 외).
- 총 건수는 필터 반영 건수. 엑셀 다운로드(US-AGG-04, US-CAT-03)는 **현재 필터 기준**임을 툴팁으로 명시.
- 스크롤은 Virtual Scroll(US-RCV-02) — 페이지네이션을 만들지 않는다.

## 5. 대시보드 컴포넌트

> 차트 일반 규칙(이 프로젝트 공통): 이중 축 금지, 시리즈 색은 엔티티에 고정(필터로 개수가
> 바뀌어도 색 유지), **발주 상태 7색은 상태 전용 — 일반 차트 시리즈 색으로 재사용 금지**,
> 2개 이상 시리즈엔 범례 필수, 값 텍스트는 텍스트 토큰 색(시리즈 색으로 글자 쓰지 않음).
> 상세 차트는 각 화면 정의서에서 다루고, 여기서는 M2·M4 대시보드의 두 위젯만 스펙 확정.

### 5.1 KPI 스탯 카드 (US-AGG-03 집계 현황)

- 카드: `surface` + `border` + radius-lg(8) + shadow-1, 패딩 16, 4개 1행 그리드(간격 16).
- 구성: 라벨(web.caption, `textCaption`) → 수치(web.numLg 24/700 tabular, `textTitle`) →
  보조행(전일 대비 등, 12px). 아이콘 장식 금지 — 숫자가 주인공.
- 집계 현황 4카드: `총 발주` `자동 승인` `승인 대기` `총 품목 수량`.
  **승인 대기 카드만** 수치를 `warning.text` 색 + 카드 클릭 시 수주 목록(승인 대기 필터)으로 이동.
  0건이면 색 강조 없음.

### 5.2 미수금 연령분석 위젯 (US-STL-02)

가맹점별 미수금을 연령 구간(0–30 / 31–60 / 61–90 / 90일+)으로 쌓은 **가로 스택 바 + 값 병기 표**.

- **연령 램프 (앰버 단일 hue 순차 — 오래될수록 진함)**:

  | 구간 | 색 | |
  |------|------|------|
  | 0–30일 | `#FDE68A` (amber-200) | 연한 단계 — 값 라벨 필수 |
  | 31–60일 | `#F59E0B` (amber-500) | |
  | 61–90일 | `#B45309` (amber-700) | |
  | 90일+ | `#78350F` (amber-900) | 최고 심각 |

  검증: 명도 단조 감소 확인, 인접 CVD ΔE ≥ 9.8. 연한 두 단계는 표면 대비 3:1 미만이므로
  **세그먼트 값 라벨 또는 우측 표 컬럼 병행이 필수**다 (색만으로 값 전달 금지).
- 스택 세그먼트 사이 **2px `surface` 갭**, 바 높이 12, 끝단 radius 2.
- 레이아웃: 행 = 가맹점. `가맹점명 | 스택 바 | 구간별 금액 4컬럼(tabular) | 합계`.
  합계 내림차순 정렬, 상단에 구간 범례 1행. 여신 한도 초과 가맹점은 행 좌측에 danger dot + 툴팁.
- 호버: 세그먼트에 툴팁(구간·금액). 이 색 램프는 연령분석 전용 — 다른 위젯에 재사용 금지.

## 6. 모달 · 슬라이드 패널

### 6.1 모달 공통

- 폭: sm 400 / md 560 / lg 720. `surface` + radius-lg + shadow-3, 오버레이 `rgba(15,23,42,0.5)`.
- 구조: 헤더(web.heading + 닫기 X) / 본문(패딩 24) / 푸터(우측 정렬 버튼 — 취소(secondary)·주액션).
- ESC·오버레이 클릭 닫기 허용. 단 **입력 중이거나 진행 중(업로드)이면 확인 후 닫기**.

### 6.2 거절 사유 모달 (US-RCV-01) — sm

```
수주 거절
──────────────────────────
가맹점 강남1호점 · ₩1,240,000 · 7건    ← 대상 요약 (web.bodyMd)
거절 사유 *                             ← textarea 3행, 필수
[한정 품목 재고 소진으로 …        ]
──────────────────────────
                [취소] [거절]           ← 거절 = danger
```

- 사유 미입력 시 거절 버튼 disabled. 거절 시 재고 복원됨을 본문에 1줄 고지 (한정 품목 포함 건만).

### 6.3 엑셀 업로드 모달 (US-CAT-02) — md

3단계 상태를 한 모달 안에서 전환: **선택 → 진행 → 결과** (플로우 상세는 D2 §4.3).

1. 선택: 드롭존(파일 드래그/클릭, `.xlsx`만) + "양식 다운로드" 링크.
2. 진행: 결정형 프로그레스 바 + `n / 1,000행 검증 중` + 취소 버튼만.
3. 결과(실패): danger 배너 "업로드 거부 — 12행 오류 (정상 988행 포함 전체 미반영)" +
   오류 테이블(행 번호 · 컬럼 · 사유, 최대 표시 후 스크롤) + [오류 리포트 다운로드] [다시 업로드].
4. 결과(성공): success 요약 + [닫기].

### 6.4 상세 슬라이드 패널 (수주/발주 상세)

- 우측에서 슬라이드 인, **w=480**, 전체 높이, shadow-3. 그리드 행 클릭으로 열림 —
  목록 컨텍스트(스크롤·필터)를 잃지 않는 것이 목적. 모달이 아닌 패널을 쓴다.
- 구조: 헤더(발주번호 + badge-md + 닫기) / 요약(가맹점·제출시각·마감회차·총액) /
  라인 테이블(품목·수량·단가·금액 — 미니 그리드, h=32 행) / 이력(상태 전이 타임스탬프) /
  푸터(해당 상태의 가용 액션 — PENDING_APPROVAL이면 승인·거절).
- REJECTED 건은 사유를 요약 영역에 danger.bg 박스로 표시 (D2 §1.2).
