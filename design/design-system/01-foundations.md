# 01. 파운데이션 (Design Tokens)

> 소유: PM/디자이너 세션. (컨셉·방향 승인 완료 — 2026-07-19)
> 기계가 읽는 값은 [tokens.json](tokens.json)이 단일 진실 공급원이다. 이 문서는 그 값의
> **의미와 사용 규칙**을 설명한다. 두 파일이 어긋나면 tokens.json이 맞다.

## 0. 토큰 3계층 구조

```
core      원시 팔레트 (hex 원본) — 화면 코드에서 직접 참조 금지
semantic  의미 토큰 (공통) — 시맨틱 4색, 발주 상태 7색. 웹·앱 동일 의미
web/app   플랫폼 토큰 — 브랜드 컬러, 타이포, 스페이싱, 라운드, 그림자
```

- 구현 규칙: 웹은 CSS 변수/Tailwind 설정으로, 앱은 `ThemeData`/상수 클래스로 옮긴다.
  **화면 코드에 hex 하드코딩 금지** — 반드시 토큰 이름을 경유한다.

### 1.2 브랜드 포인트

| 플랫폼 | 토큰 | 값 | 용도 |
|--------|------|------|------|
| Web | web.primary | `#2563EB` (blue-600) | 주요 버튼, 선택 상태, 링크 |
| Web | web.primary-hover | `#1D4ED8` (blue-700) | 호버/프레스 |
| Web | web.primary-bg | `#EFF6FF` (blue-50) | 선택 행 배경, 활성 메뉴 배경 |
| App | app.primary | `#EA580C` (orange-600) | 주요 버튼(하단 CTA), 활성 탭 |
| App | app.primary-strong | `#C2410C` (orange-700) | 텍스트 링크, 소형 텍스트에 쓰는 오렌지 |
| App | app.primary-bg | `#FFF7ED` (orange-50) | 강조 카드 배경, 활성 칩 배경 |

**접근성 규칙 (앱 오렌지)**: 흰 텍스트 on `#EA580C`는 대비 약 3.2:1 — **16px 이상 semibold의
큰 버튼에만 허용**한다. 본문 크기 텍스트나 아이콘 단독으로 오렌지를 쓸 때는 반드시
`primary-strong(#C2410C)` 이상을 쓴다.

**앰버와의 구분**: 앱 브랜드 오렌지는 '승인 대기' 앰버와 계열이 가깝다. 충돌 방지 규칙 —
오렌지는 **면(버튼·탭·칩)에만**, 앰버는 **뱃지·배너(연한 배경 + 진한 텍스트)에만** 쓴다.
같은 컴포넌트 안에서 두 색을 나란히 두지 않는다.

| 토큰 | 강조(면/아이콘) | 텍스트 | 배경 | 용도 |
|------|------|------|------|------|
| success | `#16A34A` | `#15803D` | `#F0FDF4` | 성공 토스트, 완료 확인 |
| warning | `#D97706` | `#B45309` | `#FFFBEB` | 주의 배너(마감 임박, 이월 안내) |

| info | `#2563EB` | `#1D4ED8` | `#EFF6FF` | 안내 배너 (웹 primary와 동일 hue — 의도됨) |

### 1.4 발주 상태 7색 (공통 — 이 프로젝트 디자인 시스템의 심장)

상태 머신(docs/04-domain-model 3장)의 7개 상태. **웹 수주 그리드와 앱 발주 카드에서
같은 상태는 같은 색·같은 라벨로 읽힌다.** 뱃지 형태 스펙은 D2(02-patterns)에서 정의.

| 상태 | 라벨 | dot/아이콘 | 텍스트 | 배경 | 의미 |
|------|------|------|------|------|------|
| SUBMITTED | 제출됨 | `#2563EB` | `#1D4ED8` | `#EFF6FF` | 진행 중 (마감 전 수정·취소 가능) |
| PENDING_APPROVAL | 승인 대기 | `#D97706` | `#B45309` | `#FFFBEB` | 사람의 판단 필요 |
| APPROVED | 승인됨 | `#16A34A` | `#15803D` | `#F0FDF4` | 긍정 확정 |
| REJECTED | 거절됨 | `#DC2626` | `#B91C1C` | `#FEF2F2` | 부정 종결 (사유 동반) |
| SHIPPED | 출하됨 | `#7C3AED` | `#6D28D9` | `#F5F3FF` | 물류 진행 중 |
| RECEIVED | 입고 완료 | `#0D9488` | `#0F766E` | `#F0FDFA` | 최종 완료 |
| CANCELED | 취소됨 | `#94A3B8` | `#475569` | `#F1F5F9` | 중립 종결 |

> **D1 결정 (계획서 초안에서 변경)**: RECEIVED를 "진한 그린"에서 **틸(청록)**로 변경.
> 그리드에서 APPROVED(그린)와 소형 뱃지로 나란히 놓일 때 진하기 차이만으로는 구분이
> 어렵다. 틸은 hue 자체가 달라 오독이 없고, 여전히 긍정 계열로 읽힌다.

- 상태 색은 색맹 대비를 위해 **색 단독으로 쓰지 않는다** — 항상 라벨 텍스트(또는 아이콘)를 동반한다.
- **라벨 생략 레이아웃 자체를 금지한다.** 공간이 부족하면 뱃지를 빼거나 레이아웃을 바꾼다 —
  dot 단독 표현은 어떤 화면에서도 허용하지 않는다.
  (2026-07-19 외부 리뷰 반영: 상태별 텍스처 패턴 도입 제안은 **미도입으로 결정** —
  이 금지 규칙이 유지되는 한 색 단독 상황이 발생하지 않으며, 7상태×2플랫폼 텍스처 구현
  비용 대비 효익이 없다. 라벨 생략이 불가피한 레이아웃이 실제로 등장하면 그때 재검토.)

## 5. 포맷 규칙 (공통)

| 항목 | 규칙 | 예 |
|------|------|-----|
| 금액 | `₩` + 천 단위 콤마, 소수점 없음. 표 안에서는 우측 정렬 + tabular | `₩1,240,000` |
| 수량 | 숫자 + 발주 단위 병기 (단위 생략 금지) | `3 박스` · `12 EA` |
| 날짜 | `YYYY-MM-DD`, 같은 해에는 `M월 D일(요일)` 허용 (앱) | `2026-07-19` · `7월 19일(일)` |
| 시각 | 24시간제 `HH:mm`. 마감 시각은 항상 이 형식 | `12:00 마감` |
| 상대 시간 | 24시간 이내만 허용(`n분 전`), 그 이상은 절대 날짜 | `32분 전` |
| 마감 카운트다운 | 60분 미만부터 `mm:ss`, warning 색 적용 | `29:59 남음` |
| 빈 값 | 그리드 빈 셀은 `–` (하이픈 아님, en dash) | – |

## 6. 접근성 기준선

- 텍스트 대비: 본문 4.5:1 이상, 18px+ 굵은 글씨 3:1 이상 (본 문서의 텍스트용 색은 이를 만족하는 단계로 지정했다)
- 앱 터치 타깃: 최소 48×48dp, 인접 타깃 간 8dp 이상
- 상태·피드백은 색 + 텍스트(또는 아이콘) 이중 표현 — 색 단독 금지
- 앱 스캔 화면(다크 고정): 배경 neutral-900, 텍스트 neutral-50, 인식 성공 피드백은 색+진동+소리 3중
