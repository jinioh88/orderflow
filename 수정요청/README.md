# 수정요청 — API 변경 요청함

웹/앱 개발 세션이 API 계약(`server/docs/api-spec.md`)의 변경이 필요할 때 요청 문서를 남기는 곳이다.

## 흐름

```
웹/앱 세션: 요청 문서 작성 (아래 형식) → 해당 태스크는 보류, 다른 태스크 진행
     ↓
사용자/PM: 검토 — 승인 / 반려 (status 갱신)
     ↓
서버 세션: 승인 건 반영 → api-spec.md 갱신 → 요청 문서에 처리 결과 기록 (status: 완료)
```

## 파일 규칙

- 파일명: `YYYYMMDD-<요청세션>-<주제>.md` (예: `20260715-web-login-response-role.md`)
- 요청 1건 = 파일 1개. 이 README는 수정하지 않는다.

## 문서 형식 (템플릿)

```markdown
# <요청 제목>

- status: 요청됨 | 승인 | 반려 | 완료
- 요청 세션: web | app
- 요청일: YYYY-MM-DD
- 대상 API: <METHOD> <경로> (또는 "신규")
- 관련 스토리: US-XXX-NN

## 현재 스펙

(api-spec.md의 현재 정의 — 신규 요청이면 "없음")

## 요청 내용

(원하는 변경 — 요청/응답 필드, 에러 케이스 등 구체적으로)

## 사유

(왜 필요한가 — 어떤 화면/기능이 막혀 있는가, 인수 조건 근거)

## 처리 결과 (서버 세션 기록)

(반영 내용, api-spec.md 갱신 위치, 반영일 — 완료 시 작성)
```
